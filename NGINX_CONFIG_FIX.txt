NGINX CONFIGURATION ISSUE DETECTED

The problem: Nginx is still returning 502 Bad Gateway even after restarting the server.
This means either:
1. The Node.js server is not starting on port 3001
2. Nginx configuration is not properly routing /api/ requests

IMMEDIATE NGINX FIX - Add this to Plesk "Additional nginx directives":

location /api/ {
    proxy_pass http://127.0.0.1:3001/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_connect_timeout 5s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;
}
