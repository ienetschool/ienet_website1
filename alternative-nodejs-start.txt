ALTERNATIVE: If Plesk Node.js Still Not Working

# Option 1: Manual PM2 Start (if Plesk fails)
cd /var/www/vhosts/vivaindia.com/ienet.online
pm2 start app.js --name ienet
pm2 startup
pm2 save

# Option 2: Direct Node.js Service
# Create systemd service file:
sudo tee /etc/systemd/system/ienet.service > /dev/null <<EOF
[Unit]
Description=IeNet Node.js Application
After=network.target

[Service]
Type=simple
User=ienet
WorkingDirectory=/var/www/vhosts/vivaindia.com/ienet.online
ExecStart=/usr/bin/node app.js
Restart=always
RestartSec=10
Environment=NODE_ENV=production
Environment=PORT=3000

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable ienet
sudo systemctl start ienet

# Option 3: Nginx Direct Proxy (bypass Plesk Node.js)
# Add to nginx config:
location / {
    proxy_pass http://localhost:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_cache_bypass $http_upgrade;
}

The ultra-simple app.js should work with any of these methods.
